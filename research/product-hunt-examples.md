## Example 1: Cursor
**Category:** AI-Powered Code Editor  
**Launch Context:** Product Hunt #1 in 2023  

### Positioning Strategy
- **Headline Focus:** "Write better code with an AI-powered code editor"
- **Core Value Prop:** Developer productivity (code generation, refactoring, debugging)
- **Target Audience:** Full-stack developers, engineers tired of boilerplate

### Copy Insights
- Emphasizes **workflow integration** (works with VS Code setup, Vim habits)
- Shows ROI through **time saved on routine tasks**
- Addresses switching costs: "No need to learn new tools"
- Uses technical credibility: Claude/GPT integration transparency

### Visual Strategy
- GIF-based demos showing real code editor in action (not mockups)
- Before/after comparisons: "Without AI vs With Cursor"
- Live keyboard shortcuts demonstrated
- Clean, minimal UI screenshots (dark theme, familiar to engineers)

### Pricing Model
- **Freemium:** Limited daily uses (~50 completions)
- **Pro:** $10/month or $100/year
- **Business:** Custom pricing
- **Key positioning:** "Costs less than a coffee per week"

### Launch Success Factors
1. **Product-market fit clarity:** Solved existing pain (repetitive coding)
2. **Execution speed:** High-quality integration, not rushed
3. **Community engagement:** Responded to HN/Product Hunt comments in real-time
4. **Social proof:** Engineer testimonials, before-after GIFs

## Key Learnings for RAG Template Launch

### Copy & Positioning
- **Lead with outcome, not features:** Don't say "RAG template with chunking strategies" → say "Ship RAG apps 10x faster"
- **Show the problem first:** "Retrieval is hard. Relevance is harder. We solved both."
- **Build in comparison:** "vs. Wiring RAG from scratch" or "vs. Expensive vendor solutions"
- **Use concrete examples:** Specific use cases (customer support, documentation QA, domain-specific search)

### Visual/Demo Strategy
- **Live workflow demo:** Record building a RAG app from scratch (not AI-generated, real UI)
- **Before/after:** "Without template" (100 LOC, 10 bugs) vs "With template" (20 LOC, production-ready)
- **Architecture clarity:** Diagram showing retrieval → chunking → ranking → generation flow
- **Real use case showcase:** E.g., "Built Slack bot that knows our entire codebase in 2 hours"

### Pricing Positioning
- **Freemium model recommended:**
  - **Free:** Template + docs + community support (enough to get value)
  - **Pro:** Premium templates, advanced chunking, fine-tuned ranking models ($X/month)
  - **Enterprise:** Custom retrieval strategies, SLA, white-glove setup
- **Messaging:** "Start free. Scale as you ship. Pay only for what you need."

### Audience Targeting
- **Primary:** Full-stack engineers building AI products (not pure ML engineers)
- **Secondary:** Startups building copilots, chatbots, search features
- **Tertiary:** Product teams adding AI features to existing products
- **Messaging angle:** Speed and reliability, not cutting-edge research

### Community/Content Strategy
- **GitHub as distribution:** Make template easy to fork, star, contribute
- **How-to content:** "RAG for your Notion workspace," "Slack bot with RAG," etc.
- **Transparency:** Share what works, what doesn't; be honest about limitations
- **Engagement:** Respond to issues, celebrate community projects, feature users
